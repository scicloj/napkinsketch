2  Quickstart

A minimal introduction to Plotje

Setup

Add Plotje to your deps.edn:

{:deps {org.scicloj/plotje {:mvn/version "..."}}}

Then require the API:

(ns plotje-book.quickstart
  (:require
   ;; R datasets
   [scicloj.metamorph.ml.rdatasets :as rdatasets]
   ;; Kindly -- notebook rendering protocol
   [scicloj.kindly.v4.kind :as kind]
   ;; Plotje -- composable plotting
   [scicloj.plotje.api :as pj]))

Use Clay or other Kindly-compatible tools to visualize the examples below.

Plain Data

You do not need to load a CSV – Plotje accepts plain Clojure data and coerces it into a dataset internally. A map of columns works directly:

(-> {:x [1 2 3 4 5] :y [2 4 3 5 4]}
    (pj/lay-point :x :y))

{

:layers [{:layer-type :point}]
:data

_unnamed [5 2]:

:x :y
1 2
2 4
3 3
4 5
5 4 
:mapping {:x :x, :y :y}

}

When the dataset has few columns, you can skip the column names – Plotje infers them from the dataset shape:

(-> {:x [1 2 3 4 5] :y [2 4 3 5 4]}
    pj/lay-point)

{

:layers [{:layer-type :point}]
:data

_unnamed [5 2]:

:x :y
1 2
2 4
3 3
4 5
5 4 
:mapping {:x :x, :y :y}

}

See Core Concepts for more input formats.

String column names also work – keywords are conventional but not required:

(-> {"x" [1 2 3 4 5] "y" [2 4 3 5 4]}
    (pj/lay-point "x" "y"))

{

:layers [{:layer-type :point}]
:data

_unnamed [5 2]:

x y
1 2
2 4
3 3
4 5
5 4 
:mapping {:x "x", :y "y"}

}

Inference

pj/frame declares which columns to plot without committing to a chart type. When a pipeline ends at pj/frame (no pj/lay-*), Plotje picks the chart type from the column types. Two numerical columns produce a scatter plot:

(-> iris
    (pj/frame :sepal-length :sepal-width))

A single categorical column produces a bar chart:

(-> iris
    (pj/frame :species))

A single numerical column produces a histogram:

(-> iris
    (pj/frame :sepal-length))

See the Inference Rules chapter for the full set of rules.

Multiple Layers

Use pj/frame to set column mappings for a frame, then add layers with pj/lay-*. All layers on this frame inherit the frame’s mappings. Here (pj/lay-smooth {:stat :linear-model}) adds a linear model (regression line) per group:

(-> iris
    (pj/frame :sepal-length :sepal-width {:color :species})
    pj/lay-point
    (pj/lay-smooth {:stat :linear-model}))

Dashboards

Combine multiple plots with pj/arrange:

(pj/arrange [(pj/lay-point iris :sepal-length :sepal-width {:color :species})
             (pj/lay-histogram iris :sepal-length {:color :species})]
            {:cols 2})

Export

Save a plot to SVG with pj/save. It writes the file and returns the path:

(-> iris
    (pj/lay-point :sepal-length :sepal-width)
    (pj/save "/tmp/iris-scatter.svg"))
"/tmp/iris-scatter.svg"

For PNG output, pj/save-png goes through a raster backend; see the Cookbook for other export paths.
